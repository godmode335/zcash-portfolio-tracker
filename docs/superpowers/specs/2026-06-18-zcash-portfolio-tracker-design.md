**Дата:** 2026-06-18
**Статус:** утверждён к реализации (MVP)
**Шаблон-источник:** Проект №1 «MultiPrivacyTracker» (`C:\Users\user\privacy-coin-tracker`) — наследуем архитектуру и критерии, НО с уникальным дизайном.

## 1. Резюме

Отдельный продукт — десктоп-приложение (Windows) + SEO-сайт — **только под Zcash (ZEC)**.
Та же архитектура и принципы, что в Проекте №1 (local-first, приватность,
ручной учёт, цены по API), но: одна монета, отдельный репозиторий, своё имя и
**визуально уникальный дизайн** (не перекраска №1).

## 3. Приложение (single-coin ZEC)

Наследует ядро №1, упрощённое до одной монеты:
- **Убрано:** выбор монеты, мульти-монетный список. `coins` → единственный ZEC
  (symbol ZEC, coingecko_id `zcash`).
- **Остаётся:** модель на сделках (buy/sell), холдинг ZEC по каждому портфелю,
  P&L (реализованный + нереализованный, средняя цена), несколько портфелей,
  графики (стоимость портфеля + цена ZEC), ручной ввод, цена ZEC из CoinGecko,
  локальная SQLite в AppData, фоновый планировщик цен, устойчивость к офлайну.
- Денежные суммы — Decimal; хранение TEXT; ts — INTEGER Unix.
- Дашборд показывает один актив (ZEC) на портфель: количество, средняя цена,
  цена, стоимость, P&L; график стоимости портфеля и график цены ZEC.

## 4. Сайт (Astro, как №1, без мульти-монетных страниц)

- **Лендинг** под «zcash portfolio tracker» + рабочая кнопка **Download** →
  GitHub Releases (`/releases/latest/download/<stable-asset>.exe`).
- **Блог:** 5 стартовых статей под Zcash-запросы, например:
  `track-zcash-holdings`, `zcash-wallet-portfolio-tracker`,
  `zcash-tax-tracking`, `shielded-vs-transparent-zcash`,
  `zcash-price-history-guide`. Каждая 800–1500 слов, с CTA на Download.
- **SEO:** per-page `<title>`/description/canonical/OG/Twitter; JSON-LD
  `SoftwareApplication`, `FAQPage`, `BlogPosting`, `Organization`, `WebSite`,
  `BreadcrumbList`; авто `sitemap.xml` + `robots.txt`; человекочитаемые URL;
  нулевой клиентский JS на контентных страницах.
- **Нет** `/coins/*` (одна монета) — их роль берёт лендинг.
- Хостинг: Cloudflare Pages, кастомный домен (domain-ready), приватная
  аналитика + Search Console (как №1).

## 5. Приватность и хранение (как №1)

Все данные локально; наружу — только публичные запросы цены ZEC (никаких
данных о холдингах). Данные сохраняются между запусками.

## 6. Уникальный дизайн (отличие от №1)

Проект №1: тёмная минималистичная тема, зелёный акцент, системный шрифт,
центрированный hero, простые карточки. Проект №2 — **намеренно другой язык**:

- **Палитра:** Zcash-золото `#F4B728` как акцент на тёплом «графитовом»
  фоне (тёмно-коричнево-серый, не сине-чёрный №1); продуманные светлые
  поверхности для контраста. (Точные токены подберём через `ui-ux-pro-max`.)
- **Лейаут:** не центрированный, а **редакционный/левосторонний hero** +
  **bento-grid** для фич (вместо равных карточек №1).
- **Типографика:** выраженная пара «дисплейный шрифт + чистый sans»
  (через `font-display: swap`), крупные контрастные заголовки — иной характер,
  чем системный шрифт №1.
- **Компоненты:** иная форма кнопок (pill) и карточек, иные отступы/ритм,
  собственная сетка. Цель: при сравнении со №1 — это очевидно другой продукт.
- Доступность/перформанс — те же стандарты (контраст ≥4.5:1, focus-состояния,
  reduced-motion, адаптив, оптимизированные изображения).

Конкретную дизайн-систему утвердим через skill `ui-ux-pro-max` на этапе
реализации; здесь зафиксирован вектор «уникально, не клон №1».

## 7. Тестирование / приёмка

- Юнит-тесты ядра (P&L/аллокация, db, prices) и SEO-библиотеки — как №1.
- `cargo test` зелёный; `astro build` + `astro check` без ошибок.
- Лендинг с рабочей кнопкой Download (реальный релиз `.exe`).
- Lighthouse SEO/Performance — зелёные; sitemap/robots на месте.
- Визуально сайт и приложение **отличаются** от №1 (другой стиль, не рекрас).
